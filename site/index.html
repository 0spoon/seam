Where your notes become your agents' context

A local knowledge system -- notes, projects, tasks, knowledge graph -- that every agent works inside through MCP. Plain markdown files on your machine, not a key-value store.

Memory layers aren't enough

Your agents need more than a key-value store. They need a workspace.

Memory stores forget structure

Key-value pairs and vector embeddings don't capture how ideas relate, which project they belong to, or what tasks remain.

Agents can't collaborate

Your morning session in Claude Code and your afternoon session in Cursor are isolated worlds. No shared context, no handoff.

You are the integration layer

You end up copying context between conversations, re-explaining decisions, and doing the work your tools should do for each other.

Agents work inside, not on top

Every MCP-compatible agent -- Claude Code, Cursor, Windsurf -- operates inside the same knowledge system.

Start a session

Your agent calls session_start and gets a briefing: past sessions, open tasks, relevant notes, what other agents found.

Work with shared context

Every agent reads from the same knowledge base. Findings, decisions, and research notes are stored as persistent memory.

End and handoff

session_end records what was learned. The next session -- or a different agent -- picks up where this one left off.

A workspace, not a key-value store

Notes, projects, tasks, search, a knowledge graph, and an AI that organizes it all.

MCP

MCP Server

40+ tools for sessions, memory, notes, tasks, and research labs. Works with Claude Code, Cursor, Windsurf, and any MCP client.

.md

Plain markdown

Notes are files on disk. Edit with vim, VS Code, or anything. Back up with git. Grep from the terminal. Nothing to export.

Local

100% local by default

Ollama for AI, SQLite for metadata, your filesystem for notes. Nothing leaves your machine. Switch to OpenAI or Anthropic with one config line.

AI

Semantic search

Find notes by meaning, not just keywords. Ask Seam answers questions grounded in your notes, with citations.

Graph

Knowledge graph

Wikilinks, backlinks, two-hop connections. See how ideas relate. Orphan detection finds notes that need linking.

Flex

Three interfaces

Web app, terminal TUI, and a full REST API. Same data, pick your surface. Plus WebSocket events for real-time updates.

Wire it up in five lines

Add this to your Claude Code, Cursor, or any MCP-compatible agent.

{
  "mcpServers": {
    "seam": {
      "url": "http://localhost:8080/api/mcp",
      "headers": {
        "Authorization": "Bearer <your-token>"
      }
    }
  }
}

That's it. Your agent now works inside your knowledge system -- not just reads from a memory store.

Under the hood

A single Go binary. Your markdown files are the source of truth.

  Your .md files         Seam indexes            AI connects
       |                       |                       |
       v                       v                       v
    notes/                  seam.db                 ChromaDB
    inbox/               (SQLite FTS5)            (embeddings)
    project-a/           metadata, links            semantic
    project-b/          tasks, versions              search

       +-----------+-----------+-----------------------+
                   |
             seamd (Go binary)
             |       |       |
           Web     TUI    MCP Server
         (React) (Bubble  (AI coding
                  Tea)    agents)
MissingWhat happens
OllamaAI features disabled. Markdown notes + full-text search still work perfectly.
ChromaDBNo semantic search. Full-text search and AI chat still work.
WhisperNo voice capture. Everything else works.

Get started

You'll have Seam running in under five minutes.

git clone https://github.com/0x3k/seam.git
cd seam
make build      # builds seamd (server) + seam (TUI)
make init       # interactive setup: JWT secret, data dir, LLM
make dev        # runs seamd + Vite + Chroma in parallel

What you'll need

Go 1.25+ required
Node.js 22+ for web frontend
Ollama for AI features
Docker for semantic search
