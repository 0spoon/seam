Give Your AI Agents a Second Brain

Seam exposes an MCP server that gives AI coding agents persistent, structured memory. Claude Code, Cursor, or any MCP-compatible tool can track sessions, store domain knowledge, and search across your entire knowledge base—across conversations, without losing context.

Session Management

Agents start sessions, set plans, track task progress, and record findings. Hierarchical sessions let parent agents spawn sub-agents that inherit context.

session_start("refactor-auth") -> agent gets a briefing with budgeted context from prior sessions

Knowledge Memory

Persistent domain notes organized by category that survive across sessions. An agent learns your project patterns once and remembers them forever.

memory_write("patterns", "error-handling", "This codebase uses sentinel errors...")

Context Gathering

Budgeted search across all your notes. The agent asks a question, Seam returns ranked snippets within a character budget.

context_gather( "how does auth work?", maxChars: 4000) -> returns relevant note snippets with citations

Connection Configuration

{
  "mcpServers": {
    "seam": {
      "url": "http://localhost:8080/api/mcp",
      "headers": {
        "Authorization": "Bearer <token>"
      }
    }
  }
}

Hierarchical Sessions

refactor-auth ← main agent
└── refactor-auth/analyze ← subagent A (sees parent plan)
└── refactor-auth/implement ← subagent B (sees parent plan + A's findings)

What Seam Enables

Concrete workflows that turn disconnected information into connected knowledge.

Research & Learning

Reading about distributed systems? Capture articles via URL, link concepts with [[wikilinks]], then ask Seam: "summarize everything I know about consensus algorithms"—and get a synthesis across all your notes with citations.

Personal Knowledge Base

Dump thoughts into Inbox with a keyboard shortcut. Voice-record ideas while walking. Seam transcribes, summarizes, and suggests links to related notes—turning fragments into a connected knowledge graph.

AI-Augmented Writing

Write bullet points, let AI expand them into paragraphs. Paste a long doc, get a summary. Extract action items from meeting notes. All running locally via Ollama—no cloud, no API costs.

Project Management

Organize notes by project. See your knowledge graph filtered by project. Run the review queue to find orphaned notes, untagged content, and potential connections you missed.

Architecture

Go React 19 SQLite FTS5 Ollama ChromaDB

Seam is a monolithic Go backend serving a React single-page app. Notes are stored as markdown files on disk, indexed in SQLite with FTS5 for full-text search. Embeddings are generated locally via Ollama and stored in ChromaDB for semantic search. The MCP server runs as an integrated HTTP endpoint, giving AI agents direct access to the knowledge layer.

Getting Started

Up and running in a few minutes.

Go 1.22+ Node.js 20+ Ollama (optional) ChromaDB (optional) 
# Clone the repository
git clone https://github.com/0x3k/seam.git
cd seam

# Build the frontend
cd frontend && npm install && npm run build && cd ..

# Configure
cp .env.example .env
# Edit .env with your preferred settings

# Run
go run ./cmd/seam
